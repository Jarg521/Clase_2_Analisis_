Fase Actividades
Iniciacion
  • Actividades:
    • nace la idea del producto de software
    • se identifican los clientes y segmentos del mercado
    • se entrevistan algunos clientes para identificar sus necesidades y conocer de forma general las funcionalidades que deberia tener el software
    • se hace un analisis para determinar si existen otros productos similares
    • se analiza la factibilidad del producto y la posible aceptacion por parte del usuario
planeacion
  • Actividades
    • se desarrolla un plan para determinar los recursos necesarios para implmetar el proyecto de software
    • se determina el costo aproximado del proyecto y el tiempo de desarrollo
    • dependiendo de la fecha de lanazamiento del producto se puede ajustar al cronograma
    • el plan de desarrollo del producto se actualiza de acuerdo con las necesidades del proyecto
    • se desarrolla el plan de gestion de la configuracion contiene actividades para controlar los cambios y manejar versiones de los diseños
Analisis
  • Activides:
    • se recopila la mayor cantidad de informacion posible sobre las necesidades de los clientes
    • se recopila informacion detallada sovre las funcionalidades del producto, desempeño requerido para el producto, confiabilidad, seguridad, usabilidad
    • los requerimientos generales se pueden recopilar por medio de entrevistas y lluvias de ideas con los clientes
    • los requerimientos definen que es lo que el producto de software debe hacer
Diseño
  • Actividades
    • esta dase define como el software debe ser construido para cumplir con los requerimientos definidos en la fase anterior
    • se define la arquitectura del software(division en sub-sistemas) y se doseña de forma especifica cada componente del software. todo esto se acompaña de la respectiva documentacion que es lo que conforma el documento de diseño del software

Pruebas
  • Actividades:
    • El software se prueba para determinar si cumple con los requerimientos
  • El software se prueba en tres fases
    1. cada modulo se prueba de manera independiente
    2. los modulos se van integrando para determinar si la comunicacion entre los modulos es correcta
    3. una vez se integran todos los modulos y se conforma el sistema completo, se determina si cumple con los requerimientos
Mantenimiento
  • Actividades:
    • esta fase comienza cuando el software se libera oficialemente para los clientes
        • En esta fase:
      • se reparan fallos que se presenten durante el uso por parte de los clientes
      • se implementan los requerimientos adicionales de los clientes con respecto a mejoras
      • se mejoran atributos del software como confiabilidad, desempeño, etc.
